# 逐段英文校修（Passage Revision，多版本）

目的：對一段指定文字提供「可挑選的多個改法」，讓使用者快速選一版採用，或在某版基礎上再要求微調。

## 開始前

1. 確認已讀 `papers/<paper>/style-guide.md`（若不存在，提醒可先做 `style-calibration`，但仍可用 `writing-style/style-defaults.md` 的預設基準先行）。
2. 確認**修改方向**。使用者若已說明（更正式／更簡潔／更強調貢獻／語氣更中性等），照辦；**若未說明，先用一句話問清楚再動手**，不要自行假設。
3. 確認範圍：到底改哪幾句，避免擴張到使用者沒指定的部分。

## 產出規則

- **提供 2–5 個版本**（視差異空間而定；差異不大時 2–3 版即可，不要硬湊 5 版）。
- 每個版本要是**真的不同的取向**，不只是換同義字。例如：
  - V1 — 最小改動（只修文法，保留原語序與語氣）
  - V2 — 更正式 / 更符合期刊慣例
  - V3 — 更簡潔（砍冗字、合併子句）
  - V4 — 更強調研究貢獻 / 主張更明確
  - V5 — 語氣調整（更中性客觀，或依使用者指定方向）
- 每版**附一句**說明：改了什麼、適合什麼情境。說明用繁體中文，被改的文字維持英文。
- 標出與原文相比的關鍵差異（可用「主要差異：…」一行帶過）。

## 界線（務必遵守）

- **不改研究邏輯、主張、數據、引用、論點順序。** 只動語言。
- 保留領域術語與專有名詞原樣（除非使用者要求調整）。
- 不新增原文沒有的內容或主張；不刪除實質訊息。
- 若過程中看到疑似事實／邏輯問題，**不要改**，只在最後加一句「⚠️ 觀察：…」提醒使用者。

## 輸出格式

```
【原文】
<原始英文段落>

【修改方向】<使用者指定或確認的方向>

— V1（最小改動）
<英文版本>
說明：…

— V2（更正式）
<英文版本>
說明：…

— V3（更簡潔）
<英文版本>
說明：…

（最多 5 版）

主要差異：一句話總結各版差在哪。
⚠️ 觀察（如有）：…
```

## 選定後

- 使用者指定採用某版（或「用 V2 但把第二句換成 V3 的」這類混搭）後：
  1. 產生最終定稿段落。
  2. 替換該篇 `papers/<paper>/Manuscript.md` 的對應位置。
  3. 依 `papers/_TEMPLATE/revisions/_format.md`（編修歷程紀錄格式）在 `revisions/` 存一筆（檔名 `YYYY-MM-DD-<段落標記>.md`，含 frontmatter＋原文＋採用版本＋備註），確保可回退。
- 若使用者要在某版上再修，回到「產出規則」針對該版再給 ≤5 個微調版本。

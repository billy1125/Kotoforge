# 範圍整體彙整審閱（Section Review）

目的：逐段校修容易見樹不見林。當使用者改到一個程度，這個 skill 拉高到「整段／整節」層級，找出單段看不出來的問題，並提出**方向選項**讓使用者決定，而不是一口氣替他改完。

## 何時用

- 使用者明確指定一個範圍（某一節、第 3–7 段、整個 Discussion、或全文）並要整體建議。
- 逐段編修累積到一定量後，想確認整體是否協調。

## 開始前

1. 讀 `papers/<paper>/style-guide.md`，以它為整體一致性的基準；若不存在，以 `writing-style/style-defaults.md` 的預設基準為準。
2. 確認範圍邊界（從哪到哪），以及使用者特別在意的面向（若有）。

## 審查面向（跨段落層級）

- **一致性**：術語、縮寫、拼寫（英 / 美式）、大小寫、時態在整個範圍內是否一致。
- **語氣連貫**：各段正式度是否一致，有沒有忽正式忽口語。
- **重複與冗詞**：跨段反覆出現的字詞、句型、轉折語；可精簡之處。
- **過渡與結構流暢**：段落之間銜接是否順，邏輯轉折處英文是否清楚（注意：只看「英文表達是否清楚」，不評論論證本身是否成立）。
- **分段與段落切分**：是否有過長段落把多個概念塞在一起、該拆成數段；概念區塊是否錯置；段落起承是否對應一個清楚主題。（如 §3.2 把單一長段拆成「隱私風險／演算法不透明／彙整＋列舉」3 段）
- **句構多樣性**：是否長句連發或短句連發造成節奏單調。
- **與風格指引的落差**：整體偏離 `style-guide.md` 之處。

## 產出規則

- **先給一份簡短診斷**：用條列點出這個範圍在語言層面的 3–6 個主要觀察（每點一兩句、附最小必要的例子）。
- **再提出 2–4 個整體修改方向**，讓使用者選，例如：
  - 方向 A — 統一時態與術語，最小幅度其餘不動。
  - 方向 B — 整體拉高正式度向目標期刊靠攏。
  - 方向 C — 精簡：砍冗詞、合併重複論述，縮短篇幅。
  - 方向 D — 強化段落過渡與可讀性。
  - 方向 E — 重新分段／結構重整：拆解過長段落、依概念區塊重組、平滑段落接縫（不更動論點順序與主張）。
- 每個方向說明：會動到什麼、預期效果、影響範圍大小。
- **不要在此階段直接大改全文。** 等使用者選定方向後，再交給 `passage-revision` 逐段落實（仍維持每段 ≤5 版本、可回退）。

## 界線

- 只處理英文語言層面：文法、語氣、敘述、一致性、可讀性。
- 不修改研究邏輯、論點順序、主張強度（除非使用者選的方向明確要求調整表述語氣，且不更動實質主張）。
- 疑似邏輯／事實問題 → 以「⚠️ 觀察」列出，不自行更動。

## 輸出格式

```
【審閱範圍】<範圍說明>

— 語言層面診斷
1. …
2. …
（3–6 點）

— 建議的整體修改方向（請挑選）
方向 A：… ｜ 影響範圍：小／中／大
方向 B：…
方向 C：…
方向 E（重新分段／結構重整，如適用）：…
（2–4 個）

⚠️ 觀察（如有）：…

下一步：選定方向後，我會用 passage-revision 逐段給你 ≤5 個版本落實，並保留回退紀錄。
```
